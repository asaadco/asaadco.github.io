    Then write the file with this exact content:

    ```yaml
    # .github/workflows/deploy.yml
    # Source: https://docs.astro.build/en/guides/deploy/github/
    name: Deploy to GitHub Pages

    on:
      push:
        branches: [main]
      workflow_dispatch:

    concurrency:
      group: ${{ github.workflow }}-${{ github.ref }}
      cancel-in-progress: true

    permissions:
      contents: read
      pages: write
      id-token: write

    jobs:
      build:
        runs-on: ubuntu-latest
        steps:
          - name: Checkout
            uses: actions/checkout@v4
          - name: Install, build, and upload site
            uses: withastro/action@v6
            with:
              package-manager: pnpm@latest

      deploy:
        needs: build
        runs-on: ubuntu-latest
        environment:
          name: github-pages
          url: ${{ steps.deployment.outputs.page_url }}
        steps:
          - name: Deploy to GitHub Pages
            id: deployment
            uses: actions/deploy-pages@v5
    ```

    Hard constraints — DO NOT deviate:
    - `withastro/action@v6` MUST be the build step (do NOT replace with a custom `pnpm install` + `pnpm build` sequence; the official action handles pnpm auto-detection, store caching, and artifact upload)
    - `actions/deploy-pages@v5` MUST be the deploy step
    - `package-manager: pnpm@latest` MUST be set explicitly (RESEARCH.md Pattern 2)
    - All three permissions (`contents: read`, `pages: write`, `id-token: write`) are MANDATORY for `actions/deploy-pages@v5`
    - `workflow_dispatch:` trigger MUST be present (enables manual re-run from Actions tab)
    - `concurrency` block MUST be present (prevents overlapping deploys per RESEARCH.md Pitfall #2)
    - The `deploy` job's `environment.name` MUST be exactly `github-pages` (links to the GitHub Pages environment created when "GitHub Actions" is selected as the Pages source)
